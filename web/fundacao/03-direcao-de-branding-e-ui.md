# Direção de Branding e UI do Web

## 1. Objetivo

Definir uma direção visual coerente para o produto antes do build do web.

## 2. Direção de marca recomendada

### Tese

A marca precisa transmitir:

- confiança clínica
- calma
- rigor
- organização
- tecnologia sem cara de gadget

### O que evitar

- visual hospitalar frio demais
- wellness genérico com excesso de bege e folhas
- dark mode agressivo como identidade principal
- roxo padrão de produto SaaS sem personalidade

## 3. Direção cromática recomendada

### Base

- `Petróleo profundo` como cor principal
- `Marfim quente` como cor de fundo
- `Grafite` para tipografia
- `Cobre suave` como acento
- `Verde musgo` para estados positivos

### Exemplo de paleta inicial

- Primary: `#0F4C5C`
- Surface: `#F6F1E8`
- Text: `#1F2933`
- Accent: `#C67A45`
- Success: `#3F6B61`
- Danger: `#B24A3A`

## 4. Tipografia recomendada

### Interface

- `Manrope` para UI principal

### Apoio editorial opcional

- `Source Serif 4` para alguns títulos institucionais, landing ou empty states especiais

### Regra

- dentro do app, priorizar legibilidade e consistência
- evitar misturar muitas famílias dentro das telas densas

## 5. Linguagem visual do produto

### Deve parecer

- profissional
- contemporâneo
- humano
- discreto

### Não deve parecer

- app de meditação
- fintech
- dashboard de growth
- hospital legacy

## 6. Componentes visuais que combinam com o produto

- sidebar clara e estável
- cards com borda sutil
- tabelas limpas com boa densidade
- headers objetivos
- badges sem excesso cromático
- alertas críticos bem destacados

## 7. Regra de UI para o beta

- priorizar clareza sobre ornamentação
- usar movimento com parcimônia
- reservar cor forte para ação, status e confiança
- interfaces clínicas precisam passar sensação de controle e não de caos
